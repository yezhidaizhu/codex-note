# 编辑器替换需求说明

## 背景

当前项目原本使用 Milkdown 作为 Markdown 编辑器。

在尝试支持“按功能开关控制编辑器能力”时，出现了以下问题：

- 取消最后一个功能时，设置会反选回默认值。
- 关闭某些功能后，编辑器仍然能通过手输 Markdown 语法继续使用对应能力。
- 粘贴整段 Markdown 文本时，编辑器没有按结构解析，只是普通文本。
- 为解决上述问题尝试切换到 Lexical，但替换过程不完整，导致输入法、光标、列表缩进等基础体验出现回归。

用户决定停止当前实现，交由其他人继续完成。

## 目标

替换当前编辑器实现，满足以下核心目标：

1. 编辑器以 Markdown 为核心数据格式。
2. 打开 `.md` 文件时，编辑区应按结构化样式展示，而不是纯文本框体验。
3. 粘贴整段 Markdown 内容时，应能正确解析成对应结构，而不是只保留原始文本。
4. 编辑器能力必须可配置，并且配置要真正生效，不只是隐藏按钮。
5. 输入体验必须稳定，不能破坏中文输入法、光标位置、列表缩进等基础行为。

## 必须实现的功能

### 1. Markdown 导入/导出

- 笔记内容在存储层仍然是 Markdown 字符串。
- 打开笔记时，Markdown 应解析成编辑器内部结构。
- 编辑后保存时，应稳定导出回 Markdown。
- 不允许出现内容丢失、语法吞掉、结构错乱。

### 2. 粘贴 Markdown 自动解析

- 用户从剪贴板粘贴一整段 Markdown 内容时：
  - 标题应变成标题结构
  - 引用应变成引用结构
  - 无序列表/有序列表/任务列表应变成对应列表结构
  - 代码块应变成代码块结构
  - 链接应变成链接结构
- 不能只是显示为普通文本。

### 3. 功能开关必须“真正禁用”

设置页已有 `enabledFeatures`：

- `heading`
- `bold`
- `italic`
- `blockquote`
- `bulletList`
- `orderedList`
- `taskList`
- `codeBlock`
- `link`
- `image`

需求如下：

- 关闭某功能后，对应能力必须真正失效。
- “真正失效”包括：
  - 不显示对应工具入口
  - 不响应对应快捷键/转换规则
  - 不再把对应 Markdown 语法解析成该结构
- 例如关闭 `codeBlock` 后：
  - ` ``` ` 不应再变成代码块
  - 粘贴包含代码块的 Markdown 时，不应生成代码块节点
  - 内容也不能丢失，至少要以普通文本保留下来

### 4. 全部功能可关闭

- `enabledFeatures` 允许为空数组。
- 关闭最后一个功能时，不能自动恢复默认全选。
- 设置值必须正确持久化。

### 5. 输入体验稳定

以下问题绝不能出现：

- 光标自动跳到开头
- 输入时内容被重置
- 中文输入法组合态被打断
- 输入字母被“吸走”
- 编辑中频繁重建编辑器实例

### 6. 列表基础交互

- 列表中按 `Enter` 应正常续项
- 空列表项按 `Enter` 应正常退出列表
- `Tab` 应缩进列表项
- `Shift+Tab` 应取消缩进
- 任务列表应支持相同的基础交互

### 7. 图片能力

- 保留当前项目已有图片保存逻辑：
  - 图片插入前需要确保笔记已落盘
  - 图片文件保存到 `editor.imageDirectory`
  - Markdown 中保存相对路径
- 支持图片粘贴
- 支持图片显示预览
- 若关闭 `image` 功能：
  - 禁止图片插入入口
  - 禁止图片粘贴转换

## 非功能要求

### 1. 技术约束

- 项目当前主体是 Vue 3 + Pinia + Electron。
- 新编辑器方案必须能稳定集成进现有技术栈。
- 如果选用 React 生态编辑器，需谨慎处理宿主桥接，不能引入输入法、焦点、状态同步问题。

### 2. 状态同步要求

- 编辑器内部状态和 `notesStore.activeNote.content` 同步时，必须避免循环更新。
- 不允许在每次输入后重新灌整份 Markdown 到编辑器。
- 只有在切换笔记、外部内容真正变化、或明确需要重建时，才允许重置编辑器内容。

### 3. 容错要求

- 对于已禁用但历史笔记中仍存在的 Markdown 结构，不能直接吞内容。
- 至少要安全降级成普通文本，保证用户原始内容不丢失。

## 推荐验收场景

### 场景 1：基础输入

- 打开一篇空笔记
- 连续输入英文
- 连续输入中文
- 光标位置必须稳定

### 场景 2：Markdown 粘贴解析

粘贴以下内容后，编辑器应出现结构化结果：

```md
# 标题

> 引用

- 无序项
1. 有序项
- [ ] 任务项

```js
console.log('hello')
```

[OpenAI](https://openai.com)
```

### 场景 3：功能禁用

- 关闭 `codeBlock`
- 输入 ` ``` `
- 不应生成代码块
- 也不能吞掉文本

### 场景 4：全部关闭

- 取消勾选全部功能
- 设置应成功保存
- 不应自动恢复默认值

### 场景 5：列表交互

- 创建列表
- `Enter`、`Tab`、`Shift+Tab` 行为正确

### 场景 6：图片

- 粘贴图片
- 文件成功落到图片目录
- 编辑器显示图片
- Markdown 保存相对路径

## 当前代码中可复用的部分

以下逻辑建议尽量复用，不需要重写：

- `src/state/notes.ts`
  - 笔记打开/保存
  - `saveImageAsset`
  - 聚焦请求 `editorFocusRequestId`
- `src/state/editor-settings.ts`
  - 编辑器设置读取/更新
- `electron/main/settings/service.ts`
  - `enabledFeatures` 持久化

## 交付标准

最终交付应满足：

1. `pnpm build` 通过
2. 基础输入稳定
3. Markdown 粘贴解析正常
4. 功能开关真正生效
5. 图片能力正常
6. 列表键位正常
7. 不丢内容
